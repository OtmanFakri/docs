# Project Scope

The HealthStaff project encompasses the development of a comprehensive web-based HR management system tailored for healthcare organization APTIV . This section outlines the key areas and functionalities included in the project scope.

## Key Features

<AccordionGroup>
    <Accordion title="Data Extraction and Reporting">
        Intuitive interface for easy extraction of reports, ensuring optimal accessibility to necessary information for
        HR personnel.
    </Accordion>
    <Accordion title="Employee Database">
        Maintain a comprehensive database of employee profiles, including personal information and work history, with
        flexible accessibility based on needs.
    </Accordion>
    <Accordion title="Medical Consultation Tracking">
        Allow HR administrators to easily access records of medical consultations, treatments, and prescriptions to
        monitor employee health and well-being.
    </Accordion>
    <Accordion title="Workplace Accident Management">
        Provide detailed reporting on workplace accidents, administered treatments, and sick leaves for thorough
        evaluation and accurate report generation.
    </Accordion>
    <Accordion title="Date and ID Filtering">
        Enable HR staff to easily generate reports for specific time periods and individual employees identified by
        their unique identifier (Employee ID), allowing for targeted analysis and intervention.
    </Accordion>
    <Accordion title="Role-Based Access Control (RBAC)">
        Implement RBAC to regulate access to HR reports based on user roles and permissions, ensuring data
        confidentiality and security.
    </Accordion>
</AccordionGroup>

<Note>
    The technical stack is chosen to ensure scalability, performance, and ease of maintenance. Each component plays a
    crucial role in delivering a robust and efficient HealthStaff system.
</Note>

<AccordionGroup>
    <Accordion title="Real-time Notifications">
        Firebase is integrated to provide instant, real-time notifications to users. This ensures that important
        updates, alerts, or messages are delivered promptly, enhancing user engagement and system responsiveness.
    </Accordion>
    <Accordion title="Notifications Email Integration">
        Microsoft Graph API is utilized to enable email functionality within the application. This integration allows:
        - Sending emails using the organization's email infrastructure
        - Access to user calendars and contacts (if required)
        - Seamless integration with other Microsoft 365 services
    </Accordion>
    <Accordion title="Background Task Processing">
        Celery, in conjunction with Redis, is implemented to handle background tasks efficiently. This setup allows for:
        - Asynchronous processing of time-consuming operations
        - Scheduled tasks (e.g., periodic reports, data updates)
        - Improved system performance by offloading heavy tasks from the main application thread
    </Accordion>
    <Accordion title="Version Control and Collaboration">
        Git is used as the version control system, providing:
        - Distributed development capabilities
        - Efficient branching and merging for feature development and bug fixes
        - Collaborative coding through pull requests and code reviews
        - Integration with CI/CD tools for automated testing and deployment workflows
    </Accordion>
</AccordionGroup>

<Note>
    While these features are not included in the current scope, the system architecture is designed to allow for future
    expansions and integrations.
</Note>

By clearly defining the project scope, we ensure that the HealthStaff system focuses on delivering core functionalities that address the critical needs of healthcare HR management while maintaining flexibility for future enhancements.
